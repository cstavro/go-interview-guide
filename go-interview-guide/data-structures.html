Data Structures

Core data structure questions covering arrays, strings, linked lists, stacks and queues, trees, graphs, maps, and heaps. These are baseline expectations across all engineering levels.

1. Merge two sorted arrays.

Junior

Given two sorted integer arrays, return a new sorted array containing all elements from both. Do not modify the input arrays.

Boilerplate 
Loading...

Test Data

a := []int{1, 3, 5}
b := []int{2, 4, 6}
// expected: [1, 2, 3, 4, 5, 6]

Hint

Use a two-pointer merge similar to the merge step of merge sort. Pre-allocate the result slice with length len(a)+len(b).

Follow-up

  • What is the time and space complexity?
  • How would you handle arrays with duplicate values?
  • What if one array is much larger than the other?

Context

Merging sorted sequences is a fundamental building block of merge sort, external sorting, and stream merging. The two-pointer approach is expected at every level.

Further Reading

  • pkg.go.dev: sort

2. Longest substring without repeating characters.

Intermediate

Given a string, find the length of the longest substring that contains no repeated characters.

Boilerplate 
Loading...

Test Data

"abcabcbb" // 3 ("abc")
"bbbbb"    // 1
"pwwkew"   // 3 ("wke")

Hint

Use a sliding window and a map from character to its most recent index. Expand the right pointer and shrink the left when you see a duplicate inside the window.

Follow-up

  • Can you solve it in O(n) time and O(min(n, m)) space, where m is the size of the character set?
  • How does your solution handle Unicode characters in Go?
  • What if the question asked for the actual substring instead of its length?

Context

Sliding window problems appear constantly in string parsing, network protocols, and log analysis. This is a canonical baseline for any level.

Further Reading

  • pkg.go.dev: strings

3. Merge two sorted linked lists.

Junior

Merge two sorted singly linked lists into a single sorted linked list. Return the head of the merged list.

Boilerplate 
Loading...

Test Data

list1: 1 -> 2 -> 4
list2: 1 -> 3 -> 4
// expected: 1 -> 1 -> 2 -> 3 -> 4 -> 4

Hint

Use a dummy head and compare node values one at a time. Attach the smaller node and advance its pointer.

Follow-up

  • Can you do this iteratively and recursively? What are the trade-offs?
  • How would you merge k sorted lists instead of two?
  • What if the lists are not sorted?

Context

Linked lists are less common in Go than slices, but pointer manipulation questions still show up. They test your ability to reason about references and avoid losing nodes.

Further Reading

  • pkg.go.dev: container/list

4. Valid parentheses.

Junior

Given a string containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid. An input string is valid if brackets are closed in the correct order.

Boilerplate 
Loading...

Test Data

"()"      // true
"()[]{}" // true
"(]"     // false
"{[]}"   // true

Hint

Use a stack. Push opening brackets and pop to match closing brackets. The string is valid if the stack is empty at the end.

Follow-up

  • What is the time and space complexity?
  • How would you extend this to support multiple bracket types with different priorities?
  • Can you solve it without a stack?

Context

Stack-based validation shows up in parsers, compilers, and expression evaluation. It is one of the most common baseline questions for junior and intermediate candidates.

Further Reading

  • pkg.go.dev: container/list

5. Maximum depth of a binary tree.

Intermediate

Given the root of a binary tree, return its maximum depth. The maximum depth is the number of nodes along the longest path from the root down to the farthest leaf.

Boilerplate 
Loading...

Test Data

    3
   / \
  9  20
    /  \
   15   7
// depth: 3

Hint

Recursively compute the depth of each subtree. The answer is 1 + max(depth(left), depth(right)).

Follow-up

  • Implement both recursive and iterative (BFS/DFS stack) solutions.
  • What is the time and space complexity of each?
  • How would you validate that the tree is balanced while computing depth?

Context

Tree traversal is fundamental to parsing, DOM traversal, and hierarchical data. Depth calculation is a gentle introduction to recursion and tree walking.

Further Reading

  • pkg.go.dev: container/list

6. Course schedule II.

Senior

There are a total of numCourses courses to take, labeled from 0 to numCourses-1. Some courses have prerequisites. Return a valid ordering of courses, or an empty result if it is impossible.

Boilerplate 
Loading...

Test Data

numCourses = 4
prerequisites = [[1,0],[2,0],[3,1],[3,2]]
// valid order: [0, 1, 2, 3]

Hint

Model the problem as a directed graph and use topological sorting. Kahn's algorithm (in-degree counting) or DFS with cycle detection both work.

Follow-up

  • How do you detect cycles?
  • What is the time and space complexity?
  • How would you return all valid orderings?

Context

Topological sorting appears in task scheduling, build systems, and dependency resolution. Detecting cycles in a directed graph is a senior-level expectation.

Further Reading

  • pkg.go.dev: container/list

7. Two sum.

Junior

Given an array of integers and a target value, return the indices of the two numbers that add up to target. You may assume each input has exactly one solution.

Boilerplate 
Loading...

Test Data

nums = [2, 7, 11, 15], target = 9
// expected: [0, 1]

Hint

Use a hash map to store the value you need as you iterate: complement = target - current value.

Follow-up

  • Can you solve it in one pass?
  • What if the array is sorted? Can you use the two-pointer technique?
  • How would you handle duplicates or multiple valid pairs?

Context

Hash maps are the go-to tool for O(1) lookups. Two sum is the classic introduction to trading space for time.

Further Reading

  • pkg.go.dev: maps

8. Top K frequent elements.

Intermediate

Given an integer array and an integer k, return the k most frequent elements. The order of the result does not matter.

Boilerplate 
Loading...

Test Data

nums = [1,1,1,2,2,3], k = 2
// expected: [1, 2]

Hint

Count frequencies with a map, then use a min-heap of size k or bucket sort by frequency. In Go, container/heap is the standard tool for the heap approach.

Follow-up

  • What are the time and space complexities of the heap and bucket-sort approaches?
  • How would you solve this for an unbounded stream?
  • What if you needed the elements sorted by frequency?

Context

Top-K problems are everywhere in analytics, APM, and log processing. Understanding both exact and approximate solutions is valuable at every level.

Further Reading

  • pkg.go.dev: container/heap
